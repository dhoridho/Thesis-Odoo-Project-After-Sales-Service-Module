Persistent and propagated view cache

This module will reduce Odoo worker recovery time from cache depletion about 25%.
In another words: you will get speed improvements at certain moments
If you see "Invalidating all model caches after database signaling" frequently in your log files then this module is for you.

Installation

Simply locate the module and click install. No other dependencies required.

If you have installed test module - please remove it. It will not show accurate data and simply slow down overall system.

Testing

You can test to see if real module can have benefits for you.
For testing please refer to testing module over here.

But why, when and how?

Why?

Each request to the server Odoo uses lots of data to perform response to the request.
Most of the data is cached (ormcache) or will be cached (therefore the slower request).


When?

Each time

  • Odoo server is restarted
  • Worker reaches to the end of request limit
  • Worker hits the soft/hard memory limit
  • Cache needs to be flushed to obtain consistency (product attribute changed, etc)

How?

If Odoo compiles a view - this module will save the result (hence persistent)
If other worker needs to compile the same view (and it does at some point) it can fetch already compiled view (hence propagated)
